# 各章节撰写提示词参考（内容指导）

> 本文件只回答"每章写**什么**"。所有格式硬规则（字数、加粗、表格样式、列宽、宏包、文献格式等）以 [writing-standards.md](writing-standards.md) 为唯一权威来源，本文件不重复维护。
>
> **适用范围（重要）**：本文件的章节结构/顺序仅在**无用户外部模板**（即使用回退模板 `latex-template.tex`）时作为依据。**当用户自备外部 LaTeX 论文模板时，以该外部模板的章节结构与排版为第一权威**，本文件仅在模板未规定某章写法时作内容补充参考。

## 摘要

**结构：** 概括 + 分问题 + 结尾（模板与硬约束见 writing-standards.md 第 2 节）

- 开头段：2-3 句话概括选题背景与核心研究目标
- 分问题：按子问题顺序，每个说明 ① 解决什么 ② 核心方法 ③ 关键结果（数值取自求解阶段的真实 keyMetrics）
- 结尾段：总结核心价值 / 提炼创新点 / 推广方向

## 问题重述

**结构：** 问题背景 + 核心子问题

- 分"问题背景""核心子问题"两部分
- **绝对不可照抄原文**，学术化语言改写
- 聚焦建模核心矛盾，不冗余
- 结尾为"问题分析"铺垫

## 问题分析

**结构：** 分问题 + 图文思路 + 承上启下

- 每个子问题单独小节
- 包含：相关条件 → 整体分析 → 初步方法及理由
- 思维流程图仅在图文件实际存在时引用（见 writing-standards.md 第 5 节）；流程图必须**横式（`flowchart LR`）+ 全中文 + 矢量 PDF + 统一风格**（淡紫圆角矩形+紫描边+深灰细箭头的钉死主题，主题块逐字照抄、禁自定义配色/形状、Python 兜底镜像同款；完整规则与可抄片段见 writing-standards.md 第 12 节"流程图统一风格"）
- 体现"从具体到抽象"的思维过程
- **必须描述最终采用的模型方案**（评审择优胜出的最优解法），不得描述落选解法或被否决的方案；可一句话带出"曾考虑的其他解法范式及为何选定本解法"，与"评价改进推广"章节的方法排除说明相呼应（v6 不另设解法对比子节、不出对比图）

## 数据探索与预处理（可选章）

**何时写：** 仅**数据量大/字段多/需展示清洗与特征工程**的题（如 C/E 类数据题）才单独成章；机理/优化等无数据题**不写本章**，相关内容并入"问题分析"或各问"模型建立"。

- 写则放在"问题分析"之后、"模型假设"之前，作独立 `\section`
- 内容：数据整体描述（规模/字段/缺失）→ 清洗与异常处理 → 关键分布与相关性（图表见《图表选用指南》数据探索行）→ 为后续建模铺垫
- 不滥设：内容单薄时不强行立章，避免凑版面；数值结论留到求解章

## 模型假设

6 类假设，共 5-8 条（格式见 writing-standards.md 第 6 节）：

1. 题目明确给出的假设条件
2. 排除生活中的小概率事件
3. 仅考虑核心因素，忽略次要因素
4. 模型本身要求的假设
5. 对参数形式/分布的假设
6. 与题目紧密相关、用于简化的假设

## 表格产出（现场速记 · 凡写表必看）

> 排版硬规则唯一权威仍是 [writing-standards.md](writing-standards.md) §9；此处只把**最易跑偏的三条**搬到产表现场，防止（尤其弱模型）回退到记忆里的通用左对齐定宽 longtable。

- **铁律①列居中**：每列 `>{\centering\arraybackslash}p{比例\textwidth}`，**禁裸 `p{}`/`l`/`r`/`c`**（左对齐参差是最常见错版）。
- **铁律②比例满宽**：列宽用 `比例\textwidth`、总和 = 1.04−0.04N（占满版心居中），**禁 `cm`/`pt` 绝对定宽**（会留白左靠不满宽）。
- **铁律③单底线**：`\bottomrule` 全表仅一条且只在 `\endfoot`，**表末不再重复写**（重复=双底线）。
- **优先用生成器**（省去手写踩坑）：数据驱动的表（结果表等）用 `references/make_table.py` 的 `make_longtable(df, caption, label)` 从 CSV 直接吐合规 longtable 代码再粘入。可抄骨架：

```latex
\begin{longtable}{>{\centering\arraybackslash}p{0.16\textwidth}>{\centering\arraybackslash}p{0.60\textwidth}>{\centering\arraybackslash}p{0.16\textwidth}}
  \caption{表名（中文）}\label{tab:标签} \\
  \toprule  列1 & 列2 & 列3 \\  \midrule
  \endfirsthead
  \caption[]{表名（续）} \\  \toprule  列1 & 列2 & 列3 \\  \midrule
  \endhead
  \bottomrule
  \endfoot
  内容 & ... & ... \\
\end{longtable}
```

## 符号说明

- 三列表格（符号 | 说明 | 单位），longtable 格式（遵上「表格产出」三铁律：居中 / 比例满宽 / 单底线）
- 仅收录重要变量
- **以全局统一符号表为基准**——符号与公式、图表、MATLAB 代码变量一一对应

## 模型建立

- 基于问题分析和假设，抽象为数学表达式
- 紧密结合题目，不套用模型
- 改进模型需说明改进点
- 核心公式居中，附文字解释
- **以评审择优胜出的最优解法为准**，与附录 `solve_problemN.m` 描述同一个模型

## 模型求解

- 明确软件及选择理由
- 阐述核心计算步骤
- 结果规范呈现（数值来自真实求解输出）+ 简要解读
- 过长结果放附录

### 领域转化性要求

模型求解的结论必须具有实际指导意义，而非仅追求数学上的最优：

- **输出时间窗口而非精确点**：结论涉及"最优时点"时给出置信区间（如"建议 12-14 周检测"），点估计对参数扰动过于敏感，临床上无法操作
- **风险函数连续化**：避免分段常数惩罚函数（导致优化结果聚集在跳变边界），使用连续递增风险函数（如 $R(t) = e^{\gamma(t - t_0)}$）
- **分组结果的可区分性**：若分组后各组最优解差异 < 10%（如最优时点差异 < 1 周），说明分组策略无效，应重新考虑分组变量或粒度
- **多期/多资源规划结果须输出可执行方案表**：优化得到的决策矩阵（如"地块 × 年份 × 作物 × 面积"）须整理成读者可直接照做的方案表，并配方案热力图直观呈现；只给目标值不给方案表则结论无法落地

### 方法取舍说明（v6：不另设解法对比子节）

v6 已精简：**不在求解章节设独立的解法对比子节、不生成 `compare_*` 代码或对比图**。落选方法的取舍只在"评价改进推广"章节用一句话文字体现（"曾考虑过 X 方法但因 Y 原因选择 Z"，源自 methods.md 排除方法记录），无需对比 longtable、无需对比图。求解章节专注于最优解法本身的建立与求解。

## 图表选用指南（按题型）

每问"模型建立与求解"章节须有 **3-5 张**图（不含问题分析章节的流程图），每张都"能体现建模核心"，按题型选择，避免只堆默认折线图、避免凑数无信息量的图：

| 题型 | 推荐关键图 |
|------|-----------|
| 预测/回归 | 拟合曲线 vs 实测散点叠加、残差图、交叉验证误差箱线图 |
| 分类/筛查 | ROC 曲线（标注阈值点）、混淆矩阵热力图、特征重要性条形图 |
| 优化/规划/几何 | 目标收敛曲线、**累计收益/成本收敛曲线**、解空间/参数 3D 曲面、**决策方案热力图**（地块/资源 × 作物/时段）、**几何场景示意图**（物体位置/视线/遮蔽关系）、**排样/布局示意图**（空间利用率类）、优化前后对比 |
| 随机/风险优化 | 风险-收益有效前沿、不同场景下收益分布直方图/箱线图、CVaR/VaR 标注的损失分布、风险系数 $\lambda$-收益权衡曲线 |
| 反演/拟合/机理 | 模型预测谱线 vs 实测叠加、厚度-波长等敏感性曲线、参数置信区间图 |
| 评价/决策 | 权重雷达图、得分排序条形图、权重扰动下排名变化图、**灰色关联度排序条形图** |
| 时序/信号 | 带分割竖线的**有序聚类分段图**、**小波时频图/尺度图**、去噪前后对比、自相关/趋势分解图 |
| 定价/经济 | **价格-需求拟合散点+曲线**、利润关于定价的**收益曲面/曲线**、品类×定价×补货方案表 |
| 成分数据（占比/构成） | 成分三元/三角图（ternary）、对数比变换后 PCA 双标图、成分聚类树、风化前后成分对比堆叠图 |
| 数据探索（通用） | 相关性热力图（Spearman/Pearson）、列联表卡方关联马赛克图、分组箱线图、分布直方图 + 正态 QQ 图 |
| 多组/多样本谱对比（可选进阶） | **3D 瀑布图/山脊堆叠图**（waterfall/ridgeline）：多条曲线沿"样本/测量序号"进深轴错开堆叠、按"组别"配色+半透明填充，一屏呈现"自变量 × 幅度 × 样本 × 组别"四维总览 |

- 几何/场景类问题（遮蔽、路径、布局等）**必须**配场景示意图帮助读者理解建模设定，可用 matplotlib/TikZ 绘制
- **相关面板优先合成单张多面板图**：多个相关面板要并排呈现时（多日/多模型/多场景对照、相关性+重要性等），在绘图阶段就用一张图的等大子轴画出（matplotlib `plt.subplots(1,n, constrained_layout=True)`、MATLAB `tiledlayout`），导出单个文件（PNG≥300dpi 或矢量 PDF）整体按 `\textwidth` 插入——子面板天生等大、整齐铺满。**不要把原始长宽比/面板数差异大的两张独立图硬塞进 LaTeX subfigure**（会一大一小、浮空留白）。**文字清晰度是硬指标**：面板越多字越易糊，看不清就减少每行面板数/拆图/改矢量 PDF（线条文字为主的图优先矢量 PDF，海量散点保留高 dpi PNG），绝不牺牲可读性硬塞。完整优先级、清晰度与片段见 writing-standards.md 第 1、12 节。
- **3D 瀑布图（可选，模型自行判断必要性）**：仅当数据天然是「**多条共享同一自变量轴的曲线**（≥4 条）× 还要再按组别区分」且 2D 叠放会严重互相遮挡时才用——典型为多样本/多次测量的谱线、时序、分布族总览。**触发条件不满足就别用**（如曲线 ≤3 条、或没有"组别"第二维度，普通 2D 叠加折线/分面更清楚）。它只作**定性总览**，不计入该问 3-5 张定量核心图的硬性构成；**定量比较仍须另配 2D 图**（峰位/峰高柱状或箱线、均值±标准差谱带、波长×样本热力图），因为 3D 透视会失真、不能按像素读数。绘制用 Python `matplotlib`（`Axes3D` + `PolyCollection` 半透明填充）或 MATLAB `waterfall`/`fill3`；进深轴是序号分类轴，刻度标"样本/测量序号"，配色编码组别并加图例。
- 所有图统一风格、不画 title（标题由 caption 承担）；**轴/刻度/图例/`DisplayName`/`sgtitle` 全中文，且英文字段码须经"列码→中文映射"转换，通用单位/指标/方法缩写按白名单保留**（映射表与白名单见 writing-standards.md 第 12 节）

## 模型分析与检验

- 灵敏度分析：控制变量法，观察核心参数影响（数据来自求解阶段的前置参数扫描）
- 误差分析：来源分析 + MAE/RMSE 量化（数据来自求解阶段的 5 折交叉验证）
- 模型检验：使用前检验 + 使用后检验
- 多方法交叉验证：关键结果尽量用独立的第二方法或解析特例复算互证（数值一致是强可信度证明）
- 结合图表呈现
- 指标报告要求（$R^2$/AUC/灵敏度/特异度门槛与升级措施，按题型选用）见 writing-standards.md 第 13 节

## 评价改进推广

- 优点 4 条 + 缺点 2 条，结合实际结果，客观真实
- 改进：针对缺点的可落地方案，明确升级路径（如"将 Logistic 增长模型替换为 GAM"、"引入混合效应模型吸收个体差异"），并说明预期改善
- 推广：扩展至类似问题；可点到为止地提及本题存在的其他可行建模路径（如几何解析/算法搜索/优化建模并存），体现方法视野，不展开
- 禁止"可以考虑更复杂的模型"、"需要更多数据"等无具体方向的空泛描述

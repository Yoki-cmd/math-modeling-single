# 论文写作规范（单一事实来源）

> 本文件是 math-modeling skill 全部**格式硬规则**的唯一权威来源。
> SKILL.md、workflow 脚本、math-modeling-audit、section-prompts.md 中涉及格式的规则一律以本文件为准，不得另行复制维护。
> 若与用户提供的比赛 LaTeX 模板冲突，以用户模板为准。

---

## 1. 通用格式规则

- **正文一律自然段落**，禁止分点符号（1. 2. 3. 或 ● ○ 等）
- **禁止正文加粗**：`\textbf` 仅允许出现在 ① 摘要中的"针对问题X"和数值结果 ② 问题重述中的"问题N：" ③ 模型假设的"假设N：" ④ 模型评价的"优点N：/缺点N："
- **图嵌入位置（强制）**：图一律用 `[H]`（需导言区 `\usepackage{float}`）**紧贴正文嵌入**，禁止图浮动漂到独立页或章节末
- 单图宽 `0.8\textwidth`，表宽 `\textwidth`
- **多个相关面板一行并排时，等大对齐是硬指标**（理想态见"单图多面板"片段：N 个子面板天生等大、整行铺满、无浮空留白）。按下面优先级实现，**首选合成单图，LaTeX 拼图是降级手段**：

  **① 首选·合成单张多面板图（gold standard）**——凡是 ≥2 个相关面板要并排（同一主题/同一量纲/同一序列，如"多日预测对照""多模型指标""相关性+重要性"），**在绘图阶段就用一张图的等大子轴画出来**（matplotlib `fig, axes = plt.subplots(1, n, figsize=(5.2*n, 4.2), constrained_layout=True)`、MATLAB `tiledlayout(1,n,'TileSpacing','compact')`），导出**单个图片文件**（PNG ≥300 dpi 或矢量 PDF，格式与清晰度规则见 §12），整体按 `\textwidth` 插入、用一个 `\caption`、子面板标题用 `(a)(b)(c)` 前缀写进各 `ax.set_title`。这样 axes 框天生等大、字号一致、不留浮白——这是 图8 那种效果的来源，**绝大多数并排场景都应走这条**。**注意面板数与文字清晰度的权衡**：面板越多每格越窄、字越易糊，若上页后刻度/图例/数值标注看不清，就减少每行面板数或拆图或改矢量 PDF（见 §12"图内文字必须看得清"），不靠牺牲可读性硬塞。单图多面板片段：
```latex
\begin{figure}[H]
  \centering
  \includegraphics[width=\textwidth]{figures/problemN/fig_multi.png}
  \caption{整体中文标题}\label{fig:multi}
\end{figure}
```

  **② 次选·LaTeX `subfigure` 拼独立 PNG**——**仅当**两图确需各自独立的 caption/label、或来自不同绘图后端无法合成时才用。此时等大靠两层保证，缺一不可：
    - **出图阶段统一画布**：两张图用**完全相同的 `figsize`/`Position` 与 `dpi`** 导出，长宽比一致 → 下面等宽嵌入自然等大。
    - **LaTeX 阶段按高度兜底**：长宽比仍不一致时，**识别两张图的尺寸**（PNG 读像素、PDF 读点/`pdfinfo`）算宽高比 $r=W/H$，取较扁（$r$ 大）那张在 `0.48\textwidth` 下的渲染高度作公共高度 $H_0$，两张都改用 `\includegraphics[height=H_0]`（去掉 `width`、只锁高）→ 等高不溢栏。并排时**等高比等宽更"看起来一样大"**。
```latex
\begin{figure}[H]
  \centering
  \begin{subfigure}{0.48\textwidth}\centering
    \includegraphics[width=\linewidth]{figures/problemN/fig_a.png}
    \caption{子图A中文标题}\label{fig:a}
  \end{subfigure}\hfill
  \begin{subfigure}{0.48\textwidth}\centering
    \includegraphics[width=\linewidth]{figures/problemN/fig_b.png}
    \caption{子图B中文标题}\label{fig:b}
  \end{subfigure}
  \caption{整体中文标题}\label{fig:ab}
\end{figure}
```
  （长宽比不一致时把两处 `width=\linewidth` 改为 `height=H_0`，`H_0` 取实测值如 `5.2cm`。）

  **③ 不要硬拼的情形**：原始长宽比或面板数量差异很大的两图（如"方形热力图"配"双子图条形组"、"矮条形图"配"方形散点"）**禁止**塞进各 `0.48` 的 subfigure——必然一大一小、浮空留白。改为：要么按 ① 重画成统一的单张多面板图，要么各自作 `0.8\textwidth` 独立整图分两行放，**不为省版面牺牲等大与整齐**。
- 所有图表必须被正文引用（"如图X所示""如表X所示"），图表标题/caption/坐标轴全中文
- **正文章节之间**（章与章、节与节、子节与子节）一律不加 `\newpage`，内容自然接续；唯一允许的分页是模板自带的"摘要独占一页"后的那一个 `\newpage`（紧跟 `\label{abstract:end}`），正文内部禁止任何人为断页
- 图文件不画 `title()`，标题由 LaTeX `\caption{}` 承担
- 表格格子文字一行显示，不换行
- tex 文件中只写可编译的 LaTeX 代码，不可出现模板指令占位符

## 正文页数预算（页数硬约束·决策c）

> 本节为独立的命名节（不占用 §2~§13 编号，避免错位）；SKILL.md、workflow 编译/审计阶段均引用本节。

- **有效页数上限**优先级（从高到低）：`args.pageLimit`（显式传入） > 用户模板声明的页数限制 > `30`（兜底默认）。模板页数限制由主会话在阶段一 Read 模板时正则探测（见 SKILL.md 阶段一 Step 1），算出后经 `args.pageLimit` 传入 workflow。
- 正文（摘要后、参考文献前）页数 **≤ 有效页数上限**。
- **测量法**：正文页数 = `references.tex` 起始页码 − 正文首章起始页码（由 `main.aux` 中对应标签/章节页码推算）。
- **仅当**有效上限存在且正文超限时才压缩；压缩按以下顺序逐级执行：
  1. 相邻同问相关图一行并排（**优先合成单张多面板图**，必要时再用 subfigure，见 §1）
  2. 删除重复 / 低价值图
  3. 流程图改横式（`flowchart LR`）压缩高度
  4. 精简冗长表格与段落措辞
- **压缩红线（不可逾越，保连贯性）**：禁止删除完整求解内容、禁止删核心模型，禁止任何破坏论文完整性与叙事连贯性的删改。未超限时不得为"省页"删内容。

## 2. 摘要规范

**字数硬约束**：

| 段落 | 字数 |
|------|------|
| 开头段 | ≤30字 |
| 结尾段 | ≤20字 |
| **总计** | **≤900字，不可超过** |
| 页数 | 必须=1页（编译后用 `abstract:end` 标签验证） |

**各问题字数按类型弹性分配**（总计扣除头尾后≈850字）：

| 问题类型 | 摘要占比 |
|----------|----------|
| 建模求解型（优化/预测/分类/评价） | 高（≈30-35%） |
| 数据分析型（统计/探索/可视化） | 中（≈20-25%） |
| 建议总结型（建议/总结/改进） | 低（≈10-15%） |

**整体框架**：

```latex
\begin{abstract}
本文针对【题目背景】问题，提出了基于【主要方法】的系统解决方案。

【每问从下方三类模板中选对应类型填入】

综上，本文为【领域】问题提供了高效解决方案。

\textbf{关键词：}【关键词1】；【关键词2】；【关键词3】；【关键词4】
\label{abstract:end}
\end{abstract}
```

**关键词：4-6 个**，分号分隔。关键词标记方式**以模板为准**——回退模板用摘要内 `\textbf{关键词：}` 内联形式；若用户模板定义了 `\keywords{}` 宏则用宏。`\label{abstract:end}` 必须是 `\end{abstract}` 前的最后一行（一页验证依赖它）。

**① 有数据型**（题目提供附件数据，预测/分类/优化/评价等归此类）：
```
\textbf{针对问题X：}本问是【类型】问题，需【要求】。首先，对附件数据预处理，提取【特征】，【清洗/标准化等操作】。接着，【变量筛选/降维/参数选择】。然后，采取【模型】进行【求解过程】。最后，【对比/验证/排序】，给出【输出】。结果讨论显示，\textbf{【数值结果】}，模型有效地【评价】。
```

**② 无数据型**（机理/方程/优化模型，仿真/推导/理论分析归此类）：
```
\textbf{针对问题X：}本问是【类型】问题，需【要求】。首先，分析【机理/物理过程/目标约束】，建立【方程/动力学/优化模型】。接着，设定【参数/初始条件】。然后，通过【数值方法/仿真工具/求解算法】进行求解，得到【结果】。结果讨论显示，\textbf{【数值结果】}，模型有效地【评价】。
```

**③ 建议型**（基于前文结果给出建议/策略，一笔带过）：
```
\textbf{针对问题X：}本问是建议总结问题，需【要求】。基于前文分析，从【角度1】和【角度2】出发，给出【建议数量】条建议。核心结论为\textbf{【要点】}。
```

**一页验证（编译后必做）**：
```bash
PAGE=$(grep 'abstract:end' paper/main.aux | grep -oE '\{[0-9]+\}' | head -1 | tr -d '{}')
if [ "$PAGE" -gt 1 ]; then echo "摘要超出一页（第${PAGE}页），需缩减"; fi
```
若超页，按比例缩减各问题段字数后重新编译，直到 `PAGE=1`。

**摘要禁止**：公式、模型检验内容、超字数、摘要末尾加 `\newpage`。
**摘要数值必须来自真实求解结果**（求解阶段输出的 keyMetrics），不得编造。

## 3. 引言规范

| 部分 | 格式 | 要求 |
|------|------|------|
| 问题背景 | 文字，2-3段 | 宏观背景→问题来源→当前困境→解决渴求，不分点 |
| 研究意义（可选） | 文字，1段 | 社会意义 + 科研意义 |
| 问题重述 | `\textbf{问题N：}` 每问≤2行 | 问题本质缩写，删背景，**绝对不可照抄原文**，不加分点符号 |

## 4. 总体分析规范

- 三段式结构，不画流程图
- 第1段：一句话总体表达
- 第2段：针对问题X建立了XX来解决XX（逐问串联）
- 第3段：总结各问题之间的递进关系

## 5. 问题分析规范

- 每个子问题单独小节
- 包含：相关条件 → 整体分析 → 初步方法及理由
- **仅当流程图文件实际存在时**才引用思维流程图；图不存在则纯文字论述，不得引用不存在的图
- 体现"从具体到抽象"的思维过程，不含检验和结果

## 6. 模型假设规范

- 两段式：第1段"为简化问题，本文做出以下假设："
- **itemize 环境**，每条 `\textbf{假设N：}`，≤2行
- **共 5-8 条**，覆盖 6 类：① 题目明确给出 ② 排除小概率事件 ③ 仅考虑核心因素 ④ 模型本身要求 ⑤ 参数分布假设 ⑥ 简化假设

## 7. 符号说明规范

- 三列表格（符号 | 说明 | 单位），**使用 longtable 格式**（见第 9 节）
- 仅收录重要变量，符号与公式、图表、代码变量名一致
- 以全局统一符号表为基准（多子问题符号冲突必须在写作前消解）
- **全局符号表（`notation.tex`）是唯一定义处**：建模/求解章节正文**不得重复定义符号表已有符号**（禁止再写"其中 $X$ 为……"等重复定义句）
- 公式中新出现且具全文意义的符号 → **补进全局符号表**后正文直接引用，不另立定义
- **保留（宽松档，非零说明）**：公式紧邻的**非定义性物理解读句**（取值范围/边界行为/检验结论，如"$\text{CR}<0.10$ 通过一致性检验""当 $A\to\infty$ 时 $\eta\to1$ 趋于饱和"）可少量保留以提升可读性

## 8. 模型评价/改进/推广规范

- **优点 4 条 + 缺点 2 条**，按模型写不按问题写，优点多于缺点
- itemize 格式，`\textbf{优点N：}` / `\textbf{缺点N：}`，每条≤3行
- 改进与推广：各一段自然段落，不分点
- 改进必须针对具体缺陷且可操作（如"将 Logistic 替换为 GAM"），禁止"可以考虑更复杂的模型"等空泛表述

## 9. 表格规范（全论文统一）

**所有表格**（符号说明、算法对比、模型结果、检验分析等一切表格）统一使用 `longtable` + `>{\centering\arraybackslash}p{}`，总宽=`\textwidth`，各列居中可跨页。

**列宽公式（强制）**：各 `p{比例\textwidth}` 比例总和 = **1.04 − 0.04 × N**（N 为列数）：

| 列数 | 比例总和 | 示例分配 |
|------|----------|----------|
| 2列 | 0.96 | `{0.18+0.78}` |
| 3列 | 0.92 | `{0.16+0.16+0.60}` |
| 4列 | 0.88 | `{0.12+0.12+0.12+0.52}` |
| 5列 | 0.84 | `{0.12+0.18+0.18+0.18+0.18}` |
| 6列 | 0.80 | `{0.10+0.14+0.14+0.14+0.14+0.14}` |
| 7列 | 0.76 | `{0.10+0.11×6}` |
| 8列 | 0.72 | `{0.09×8}` |

**标准模板**：

```latex
\begin{longtable}{>{\centering\arraybackslash}p{0.18\textwidth}>{\centering\arraybackslash}p{0.78\textwidth}}
  \caption{表名（中文）}
  \label{tab:标签} \\
  \toprule
  列1 & 列2 \\
  \midrule
  \endfirsthead
  \caption{表名（续）} \\
  \toprule
  列1 & 列2 \\
  \midrule
  \endhead
  \bottomrule
  \endfoot
  内容 & ... \\
\end{longtable}
```

规则：`\caption{}` 在 `\label{}` 之前，`\label{}` 后紧跟 `\\`；`\endfirsthead/\endhead/\endfoot` 固定结构；`\endfoot` 含 `\bottomrule`，表末不再重复写；格子文字一行显示；编译后检查 Overfull。

## 10. 参考文献规范

- **必须真实存在、可检索**。禁止保留模板示例文献，**禁止通过翻译/音译英文文献伪造"中文文献"**（不可检索的条目属于虚假引用，学术诚信红线）
- **语言策略**：优先选用真实存在的中文文献（中文教材、综述、核心期刊论文）；该主题确无合适中文文献时保留英文原文按 GB/T 7714-2015 外文文献格式著录
- 标签格式 `作者姓氏+年份+关键词`（如 `\bibitem{behzadian2012topsis}`）
- GB/T 7714-2015 格式，纯文本一行写完，严禁 `\emph{}` `\textbf{}` `\textit{}` `\underline{}`，删除 DOI 号
- **数量**：8-15 条
- **引用要求**：每条 `\bibitem{}` 必须在正文中被 `\cite{}` 引用，一一对应；引用位置覆盖模型建立、模型求解、模型检验、引言

## 11. 附录规范

- 附件说明表，两列，不写运行环境/硬件环境
- 求解代码通过 `codebox` 环境内联展示。**文件名后缀随实际求解环境**：MATLAB 用 `.m`，回退 Python 时为 `.py`（本节示例以 `.m` 书写，Python 项目对应替换）。**导言区仅需**：

```latex
\usepackage[most]{tcolorbox}

\newtcblisting{codebox}[1][]{%
  listing only,
  colback=gray!5,
  colframe=black!70,
  fontupper=\scriptsize\ttfamily,
  boxrule=0.8pt,
  arc=2pt,
  left=4pt, right=4pt, top=2pt, bottom=2pt,
  breakable,
  #1
}
```

（`[most]` 选项已自带 listings 支持，**无需**额外 `\usepackage{listings}` / `\usepackage{xcolor}`）

- 附录结构：第一个代码框必须是 `common_utils.m`（或 `.py`，公共依赖），之后按问题编号排列各 `solve_problemN.m`（或 `.py`，最优解法求解代码）。v6 不生成 `compare_*` 对比代码，附录不含对比代码框
- 每个代码框标题格式：`title={\texttt{文件名} --- 功能说明}`（文件名中 `_` 需转义为 `\_`）
- 代码为完整可运行版本（非简化伪代码），含注释；`listing only` 模式下代码内容无需 TeX 转义

## 12. 图表与数据规范

- **流程图**：依次尝试 mermaid-cli（`npx -y @mermaid-js/mermaid-cli`，会自动下载）与 Python（matplotlib/graphviz）；两种方法尝试下载后均不可用或都生成失败则不引用该图。**必须输出矢量 PDF（非 PNG），规避缩放失真**：mermaid-cli 用 `-o figures/problemN/flow.pdf --pdfFit`（输出后缀决定格式，`--pdfFit` 裁去白边，中文由内部 Chrome 渲染；各问思维流程图统一落于 `figures/problemN/flow.pdf`，全局技术路线图落于 `figures/roadmap.pdf`）；Python 兜底同样产矢量 PDF（graphviz `-Tpdf` 或 matplotlib `savefig('....pdf')`）。按横图宽高比嵌入、宽度 `0.9–1.0\textwidth`，不拉伸变形。
- **流程图统一风格（强制·钉死，禁止各模型自由发挥）**：默认采用「淡紫圆角矩形 + 紫色细描边 + 深灰细箭头 + 横向 LR」这一套（即 mermaid 默认淡紫主题）。统一规则：
  1. **方向**：横式 `flowchart LR`（省纵向版面，禁止默认竖式 TD）。
  2. **主题块逐字照抄**（写在每个 mermaid 源码首行，钉死配色/字体，消除版本默认差异）：
```
%%{init: {'theme':'base','themeVariables':{'fontFamily':'Microsoft YaHei','primaryColor':'#ECECFF','primaryBorderColor':'#9370DB','primaryTextColor':'#1a1a1a','lineColor':'#333333','fontSize':'15px'}}}%%
flowchart LR
  A["附件数据<br/>多变量时序"] --> B["数据清洗<br/>重构·插值·去异常"]
  B --> C["问题一<br/>因子筛选+多模型预测"]
  B --> D["问题二<br/>异质时滞ARX动态模型"]
  C --> E["求解与训练"] 
  D --> E --> F["模型检验"] --> G["结论"]
```
  3. **节点形状统一为圆角矩形** `id["文字"]`，多行用 `<br/>`；**禁止混用**菱形/圆/体育场形等其它形状，除非判定/分支语义确有必要。
  4. **禁止自定义 `classDef`/`style`/逐节点上色**——一律走上面主题块的统一配色；确需高亮关键节点时，最多新增**一种**强调色并在全篇所有流程图中保持一致。

- **图表多样性**：同一子问题的图种应多样且每张有建模价值（避免全是折线、避免凑数无信息量的图）；按 section-prompts.md《图表选用指南》按题型选图

- **数据真实性**：必须使用真实计算数据，不允许编造；同一变量在不同位置（正文/摘要/表格/CSV）数值必须一致；数值保留 2-4 位小数

## 13. 模型检验规范

检验维度**按题型选用**，不对不适用的模型强套指标：

- 灵敏度分析：控制变量法，至少 2 个核心参数（所有题型通用）
- 模型检验：使用前检验 + 使用后检验（所有题型通用）
- 【预测/回归】报告 $R^2$ 与调整 $R^2$（$R^2<0.6$ 须说明原因并给替代方案，如 GAM/NLME）+ 误差分析 MAE/RMSE，优先 5 折交叉验证
- 【分类】报告 AUC、灵敏度、特异度（AUC<0.8 须说明升级措施，如 XGBoost/LightGBM）；不得仅凭 p 值显著声称"拟合良好"
- 【优化/规划/几何】**不报 $R^2$/AUC**：报目标收敛曲线、多初值/多次重启的解稳定性、与 baseline 或理论界对比、数值积分步长 $\Delta t$ 的收敛性、约束满足验证；**用元启发式（遗传/差分进化/退火等）求解可线性化的 LP/MILP 时，必须报告最优性间隙**——以 LP 松弛上界或精确求解器（LINGO/Gurobi/intlinprog/PuLP）的解为标尺，给出 $\text{gap}=(\text{上界}-\text{启发式解})/\text{上界}$，gap 过大须加大迭代或改用精确求解器
- 【随机/风险优化】（随机规划、CVaR/VaR、鲁棒优化、益损值矩阵）：报告风险-收益权衡（有效前沿）、置信水平 $\alpha$ 与风险厌恶系数 $\lambda$ 的灵敏度、蒙特卡洛场景数 $N$ 增大时目标值的收敛、与确定性模型（用期望参数求解）的解对比；益损值矩阵须同时给 ≥2 种决策准则结果并说明取舍
- 【反演/拟合/机理】报拟合 RMSE 与 $R^2$、预测曲线与实测**叠加对比图**、关键参数的**置信区间**、病态性/正则化讨论
- 【微分方程/动力学机理】参数恒定时求解析解（特征根/拉普拉斯变换）、参数时变时求数值解（Runge-Kutta/`ode45`），**恒定情形下解析解与数值解必须数值吻合**（机理题最强的求解正确性验证）；并报守恒量核对、数值积分步长收敛性
- 【成分数据】（各分量和为定值的占比/浓度/构成数据）：建模前先做**闭合校验**（分量和≈100%，越界样本剔除/归一化）与**对数比变换**（CLR/ILR，零值先做零替换），**禁止对原始百分比直接算 Pearson 相关或做 PCA**（闭合约束导致伪相关）；报告变换前后结论一致性、零替换敏感性、预测成分的闭合性与实测对比
- 【评价/决策】权重扰动稳健性、排序稳定性（如权重 ±10% 排名是否反转）
- 【聚类/分段/信号】聚类数 $K$、有序分割段数、小波基与分解层数等**结构参数禁止拍脑袋**：聚类按轮廓系数/肘部法，有序分割按"段数-段内误差"拐点，小波按信号特征说明选择依据；分段结果须有业务可解释性，并与独立方法（如小波突变点 vs Fisher 分割点）互证
- 【时序】预测建模前先用 R/S 分析（Hurst 指数）判断长记忆性：$H\approx0.5$ 接近随机游走时长期预测不可靠，须说明并谨慎外推
- 医疗筛查等不对称代价场景：在灵敏度 ≥ 95% 硬约束下最大化特异度，禁止直接最大化 F1/准确率
- 相关性分析：数值变量注明 Pearson/Spearman 与显著性；**类别变量×类别变量**的关联（如"是否风化"与"类型/颜色/纹饰"）须用**卡方独立性检验**（小样本或期望频数<5 改 Fisher 精确检验/Yates 校正），不可用 Pearson/Spearman；多变量/多分组同时检验须做 Bonferroni 或 FDR 多重比较校正
- **不确定性量化**（测量误差显著或建议型结论必做）：将输入误差（如 $c\sim N(\mu,\sigma^2)$）传播到关键结论，用解析或蒙特卡洛/Bootstrap 给出结论的置信区间/鲁棒区间，而非单点
- **多方法交叉验证**（强可信度证明）：关键结果尽量用独立的第二种方法（或解析特例/极限情形）复算，数值一致即互相印证——如同一动力学量用 Runge-Kutta、`ode45`、解析解三种求解器得到量级一致的结果（2022A 波浪能三篇分别用三法均得最大功率 229~230W），或定位问题用代数解与计算几何解互证，是最有力的正确性佐证
- **数量级合理性核对**（必做底线）：关键结果须与国标/经验值/解析估计对照量级；若独立方法或不同子模型的同一结论**数量级不一致**（如同一抽样方案一处得 $n<100$、另一处得 $n>1000$），**必有一处建模有误，须定位并修正错误源**，绝不能把互相矛盾的两个值并列写进论文。**优化题专项**：目标值（利润/成本等）须以 LP 松弛上界或精确解为标尺自检，若元启发式解显著低于上界（gap 大）即判未收敛而非"模型差异"——同一道大规模 MILP 用不同算法得数倍悬殊目标值，几乎必是欠优解未收敛所致

## 14. 版式硬错判定（Phase D Layer1/2 可执行检测的唯一权威）

渲染后才暴露的版式硬错在 `.tex` 源码里看不出，须在终验编译成功后按下列确定性检测判级（模型无关，永远执行）：

- `Missing character: There is no ... in font`（main.log）→ **critical**：CJK 缺字/乱码，零容忍，必修复重编。
- `Overfull \hbox (>15pt)` → **critical**；`(>10pt)` → **high**：表格/公式/文本越界出版心。
- 非封面页 `pdftotext -f N -l N main.pdf -` 文本字节≈0 → **critical**：疑似空白/缺页。
- `pdffonts main.pdf` 未见 CJK 字体嵌入 → **high**：字体未嵌入，PDF 在他机可能乱码。
- **无视觉能力时上述阈值收紧**：`Overfull > 10pt` 或任一 `Missing character` 即判 fail，不放过。

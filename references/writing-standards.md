# 论文写作规范（单一事实来源）

> 本文件是 math-modeling skill 全部**格式硬规则**的唯一权威来源。
> SKILL.md、workflow 脚本、math-modeling-audit、section-prompts.md 中涉及格式的规则一律以本文件为准，不得另行复制维护。
> 若与用户提供的比赛 LaTeX 模板冲突，以用户模板为准。

---

## 1. 通用格式规则

- **页面边距统一**：版心上下左右边距一律 **2.5cm**（`\usepackage[top=2.5cm,bottom=2.5cm,left=2.5cm,right=2.5cm]{geometry}`），全文一致，摘要页与正文页不得有边距差异；首页用 `titling` 的 `\setlength{\droptitle}{-2em}` 去掉 `\maketitle` 顶部默认留白，使首页正文起始线与后续页对齐
- **正文一律自然段落**，禁止分点符号（1. 2. 3. 或 ● ○ 等）
- **正文一律禁止加粗**：章节正文（含摘要"针对问题X"/数值结果、问题重述"问题N："、模型假设"假设N："、模型评价"优点N：/缺点N："）**一律不得手写 `\textbf`/`\bfseries`**。全文唯一允许的加粗是**章节标题**——由预导言 `\titleformat` 的 `\bfseries` 统一承担（一/二/三级标题加粗，三级标题为 `\normalsize\bfseries`，**不加 `\itshape`**）；图表题号（caption label）不加粗（`\captionsetup{font=small}`，不带 `labelfont=bf`）。即：sections/*.tex 内不得出现任何 `\textbf`/`\bfseries`
- **全文禁止斜体（强制·钉死）**：①三级标题用 `\normalsize\bfseries`（**绝不加 `\itshape`/`\slshape`**——中文无真正意大利体，斜体是机械倾斜、排版观感差，学术中文论文不宜用）；②正文禁止 `\emph`/`\textit`/`\itshape`/`\slshape`（含**用 `\emph` 当小标题或强调**——`\emph` 在正文渲染为斜体，既违反"正文不加粗"精神又引入斜体）。需要小标题层级时用 `\subsubsection`，需要强调时改用**措辞**而非字体（如把"在……方面，……"作段首句，而非斜体小标题）。审查检测：`grep -rnE '\\emph|\\textit|\\itshape|\\slshape' sections/*.tex` 命中即违规；并核 main.tex 的 `\titleformat{\subsubsection}` 不含 `\itshape`/`\slshape`
- **图嵌入位置（强制）**：图一律用 `[H]`（需导言区 `\usepackage{float}`）**紧贴正文嵌入**，禁止图浮动漂到独立页或章节末
- 单图宽 `0.8\textwidth`，表宽 `\textwidth`
- **多个相关面板一行并排时，等大对齐是硬指标**（理想态见"单图多面板"片段：N 个子面板天生等大、整行铺满、无浮空留白）。按下面优先级实现，**首选合成单图，LaTeX 拼图是降级手段**：

  **① 首选·合成单张多面板图（gold standard）**——凡是 ≥2 个相关面板要并排（同一主题/同一量纲/同一序列，如"多日预测对照""多模型指标""相关性+重要性"），**在绘图阶段就用一张图的等大子轴画出来**（matplotlib `fig, axes = plt.subplots(1, n, figsize=(5.2*n, 4.2), constrained_layout=True)`、MATLAB `tiledlayout(1,n,'TileSpacing','compact')`），导出**单个图片文件**（PNG ≥300 dpi 或矢量 PDF，格式与清晰度规则见 §12），整体按 `\textwidth` 插入、用一个 `\caption`、子面板标题用 `(a)(b)(c)` 前缀写进各 `ax.set_title`。这样 axes 框天生等大、字号一致、不留浮白——这是 图8 那种效果的来源，**绝大多数并排场景都应走这条**。**注意面板数与文字清晰度的权衡**：面板越多每格越窄、字越易糊，若上页后刻度/图例/数值标注看不清，就减少每行面板数或拆图或改矢量 PDF（见 §12"图内文字必须看得清"），不靠牺牲可读性硬塞。单图多面板片段：
```latex
\begin{figure}[H]
  \centering
  \includegraphics[width=\textwidth]{figures/problemN/fig_multi.png}
  \caption{整体中文标题}\label{fig:multi}
\end{figure}
```

  **② 次选·LaTeX `subfigure` 拼独立 PNG**——**仅当**两图确需各自独立的 caption/label、或来自不同绘图后端无法合成时才用。此时等大靠两层保证，缺一不可：
    - **出图阶段统一画布**：两张图用**完全相同的 `figsize`/`Position` 与 `dpi`** 导出，长宽比一致 → 下面等宽嵌入自然等大。
    - **LaTeX 阶段按高度兜底**：长宽比仍不一致时，**识别两张图的尺寸**（PNG 读像素、PDF 读点/`pdfinfo`）算宽高比 $r=W/H$，取较扁（$r$ 大）那张在 `0.48\textwidth` 下的渲染高度作公共高度 $H_0$，两张都改用 `\includegraphics[height=H_0]`（去掉 `width`、只锁高）→ 等高不溢栏。并排时**等高比等宽更"看起来一样大"**。
```latex
\begin{figure}[H]
  \centering
  \begin{subfigure}{0.48\textwidth}\centering
    \includegraphics[width=\linewidth]{figures/problemN/fig_a.png}
    \caption{子图A中文标题}\label{fig:a}
  \end{subfigure}\hfill
  \begin{subfigure}{0.48\textwidth}\centering
    \includegraphics[width=\linewidth]{figures/problemN/fig_b.png}
    \caption{子图B中文标题}\label{fig:b}
  \end{subfigure}
  \caption{整体中文标题}\label{fig:ab}
\end{figure}
```
  （长宽比不一致时把两处 `width=\linewidth` 改为 `height=H_0`，`H_0` 取实测值如 `5.2cm`。）

  **③ 不要硬拼的情形**：原始长宽比或面板数量差异很大的两图（如"方形热力图"配"双子图条形组"、"矮条形图"配"方形散点"）**禁止**塞进各 `0.48` 的 subfigure——必然一大一小、浮空留白。改为：要么按 ① 重画成统一的单张多面板图，要么各自作 `0.8\textwidth` 独立整图分两行放，**不为省版面牺牲等大与整齐**。
- 所有图表必须被正文引用（"如图X所示""如表X所示"），图表标题/caption/坐标轴全中文
- **正文章节之间**（章与章、节与节、子节与子节）一律不加 `\newpage`，内容自然接续；唯一允许的分页是模板自带的"摘要独占一页"后的那一个 `\newpage`（紧跟 `\label{abstract:end}`），正文内部禁止任何人为断页
- 图文件不画 `title()`，标题由 LaTeX `\caption{}` 承担
- 表格格子文字一行显示，不换行
- **标题间距（`\titlespacing*`，预导言统一设置）**：二级标题段前 3pt、段后 0（`\titlespacing*{\subsection}{0pt}{3pt}{0pt}`）；三级标题段前 6pt、段后 3pt（`\titlespacing*{\subsubsection}{0pt}{6pt}{3pt}`），与二级标题拉开层次
- tex 文件中只写可编译的 LaTeX 代码，不可出现模板指令占位符

## 1a. 公式排版规则

- **所有行间公式一律居中且自动编号**（编号右对齐，为 LaTeX `equation`/`align` 默认行为）：单式用 `equation`，多行用 `align`/`aligned`，方程组用 `cases`（外层套 `equation` 以获得编号）
- **禁止无编号行间公式**：不得使用 `\[ ... \]`、`$$ ... $$`、`equation*`、`align*`、`gather*` 等无编号环境（即使该式未被正文 `\ref` 引用，也保留自动编号）
- 每个行间公式紧跟 `\label{eq:xxx}`；正文用 `\eqref{}` 或 `式~\ref{}` 引用
- 行内量用 `$...$`，不承载独立公式编号
- **示性/指示函数禁用 `\mathbb{1}`（强制·字形护栏）**：示性函数/指示函数一律用 `\mathds{1}`（需导言区 `\usepackage{dsfont}`，回退模板已预置）或 `\mathbbm{1}`（需 `\usepackage{bbm}`）；**禁止 `\mathbb{1}`**——`\mathbb`（amssymb/amsfonts）**仅对大写字母 A–Z 定义了 blackboard-bold 字形，对数字 `1` 无字形**，会**静默回退成破碎/错位的错误字符且不报编译错误**（0 Error），纯确定性检测（grep 公式、查 .log Error）无法捕获，只有人眼看 PDF 才发现——属"编译通过≠排版正确"盲区。示性函数在数据型/概率型题（计数=对指示变量求和、期望=指示函数积分）极常用，踩坑概率高。**审查检测**：`grep -rnE '\\mathbb\{[0-9]\}' sections/*.tex` 命中即违规，改 `\mathds{1}`。
- **审查检测**（命中即违规，用 fixed-string 避免 grep 转义坑）：`grep -rnF -e '\[' -e '$$' -e '\begin{equation*}' -e '\begin{align*}' -e '\begin{gather*}' sections/*.tex`

## 正文页数预算（页数硬约束·决策c）

> 本节为独立的命名节（不占用 §2~§13 编号，避免错位）；SKILL.md、workflow 编译/审计阶段均引用本节。

- **页数为区间约束 `[PAGE_MIN, PAGE_MAX]`**（不只是上限）：
  - `PAGE_MAX` 优先级（从高到低）：`args.pageLimit`（显式传入） > 用户模板声明的页数限制 > `30`（兜底默认）。
  - `PAGE_MIN` 优先级：用户显式指定 > `25`（兜底默认）。`PAGE_MIN` 不得超过 `PAGE_MAX`；若模板/用户给出的上限 < 25，则取 `PAGE_MIN=PAGE_MAX`（退化为单点约束）。
  - 模板页数限制由主会话在阶段零 Read 模板时正则探测（见 SKILL.md 阶段零），算出后传入。
- **"正文"范围定义（强制）**：摘要页之后 → **参考文献之前**。**不含**参考文献，**不含** `\appendix` 之后的附件清单与全部代码框。即附录代码再长也不计入正文页数。
- **目标**：`PAGE_MIN ≤ 正文页数 ≤ PAGE_MAX`（默认 25–30 页）。
- **测量法**：正文页数 = `page(sec:refstart) − page(sec:bodystart) + 1`。其中 `\label{sec:bodystart}` 置于正文首章（问题重述）`\section` 后、`\label{sec:refstart}` 用 `\phantomsection\label{}` 置于参考文献起始处（两锚点已写入回退模板，用户外部模板缺失时由写作阶段补插）；页码从 `main.aux` 的 `\newlabel` 推算。
- **超上限（> PAGE_MAX）时压缩**，按顺序逐级执行：
  1. 相邻同问相关图一行并排（**优先合成单张多面板图**，必要时再用 subfigure，见 §1）
  2. 删除重复 / 低价值图
  3. 流程图改横式（`flowchart LR`）压缩高度
  4. 精简冗长表格与段落措辞
- **低于下限（< PAGE_MIN）时扩充**，按价值优先逐级补充：
  1. 补充模型推导/公式中间步骤与物理解释
  2. 加深结果解读、灵敏度分析与误差分析的讨论
  3. 补有建模价值的图表（在每问 3–5 图上限内）
  4. 充实问题分析与模型检验的论证
  - **扩充红线**：禁止灌水——不得加无信息量的图、不得重复论述同一内容、不得拉长无关背景或堆砌套话来凑页数。
- **压缩红线（不可逾越，保连贯性）**：禁止删除完整求解内容、禁止删核心模型，禁止任何破坏论文完整性与叙事连贯性的删改。区间内不得为"省页/凑页"增删内容。

## 2. 摘要规范

**字数硬约束**：

| 段落 | 字数 |
|------|------|
| 开头段 | ≤30字 |
| 结尾段 | ≤20字 |
| **总计** | **≤900字，不可超过** |
| 页数 | 必须=1页（编译后用 `abstract:end` 标签验证） |

**各问题字数按类型弹性分配**（总计扣除头尾后≈850字）：

| 问题类型 | 摘要占比 |
|----------|----------|
| 建模求解型（优化/预测/分类/评价） | 高（≈30-35%） |
| 数据分析型（统计/探索/可视化） | 中（≈20-25%） |
| 建议总结型（建议/总结/改进） | 低（≈10-15%） |

**禁用 `\begin{abstract}` 环境（强制·问题B修复）**：article 文类的 `abstract` 环境 = `\small` + `quotation`（双侧缩进），会让摘要页**字号更小、左右边距比正文宽**，与正文页观感不一致（即"摘要页和正文页边距不同"的真正根因——与页面 `geometry` 无关，geometry 改了也治不好）。**一律改用自定义等宽等字号块**：居中"摘要"标题 + `\normalsize` 整宽段落。审查检测：`grep -n 'begin{abstract}' sections/abstract.tex` 命中即违规（用户外部模板自带 abstract 环境则以用户模板为准）。

**整体框架**（**全程不加粗**：摘要内禁止任何 `\textbf`，含"针对问题X"、数值结果、关键词均不加粗）：

```latex
% 不用 abstract 环境（避免 quotation 缩进+\small 致摘要页边距/字号异于正文）
\begin{center}{\Large 摘要}\end{center}
\vspace{0.5em}

\noindent 【开头段：直接点题 + 一句话核心思路，≤30字。不写"提出了基于…的系统解决方案"这类套话】

针对问题X，……（每问从下方三类模板挑句式，注意各问连接词与句式不要雷同）

【结尾段：落到一句具体结论或最有辨识度的发现，≤20字。不写"综上，本文为…提供了高效解决方案"】

\vspace{0.5em}
\noindent 关键词：【关键词1】；【关键词2】；【关键词3】；【关键词4】
\label{abstract:end}
```

**关键词：4-6 个**，分号分隔，**不加粗**。关键词标记方式**以模板为准**——回退模板用 `\noindent 关键词：` 内联形式（无 `\textbf`）；若用户模板定义了 `\keywords{}` 宏则用宏。`\label{abstract:end}` 必须是摘要块的最后一行、紧接其后的 `\newpage`（一页验证依赖它）。

### 去 AI 味（强制，问题7修复）

机械的"首先→接着→然后→最后→结果显示"四段式 + 套话首尾句是 AI 味的主要来源。写摘要时遵守：

1. **删除固定四段式骨架**：不要每问都"首先…接着…然后…最后…"。同一篇摘要里**各问的连接词与句式必须轮换**（如一问用"针对…，构建…，得到…"，另一问用"考虑到…，引入…，据此…"），不得三问同构。
2. **句长有长短变化**，可把"方法+关键结果"揉进 1–2 个长句，避免每句等长的排比流水账。
3. **套话黑名单**（禁用）：`综上所述`、`高效解决方案`、`提供了……决策依据/有力支撑`、`具有重要的现实意义`、`随着……的发展`、`本文提出了基于……的系统解决方案`。
4. **结尾句**必须落到**具体数值结论或具体发现**，不是泛泛价值表态。
5. 数值结果照常写明（来自真实 keyMetrics），但**不加粗**。

> **下面三类模板只提供"要素清单"，不是逐字照抄的句式**——写时按上面 5 条重组语言，避免与模板措辞雷同。

**① 有数据型**（题目提供附件数据，预测/分类/优化/评价等归此类）——要素：数据预处理与特征 → 变量筛选/降维/参数选择 → 模型与求解 → 对比/验证/排序与输出 → 关键数值结果与简短评价。示例（仅参考，需改写避免雷同）：
```
针对问题X，本问需【要求】。在对附件数据【清洗/标准化】并提取【特征】后，通过【变量筛选/降维】确定输入，采用【模型】完成【求解过程】，并以【基线】为对照、以【验证方式】检验。【数值结果】，表明模型【评价】。
```

**② 无数据型**（机理/方程/优化模型，仿真/推导/理论分析归此类）——要素：机理/目标约束分析 → 建立方程/动力学/优化模型 → 参数/初值设定 → 数值方法/仿真求解 → 关键数值结果与评价。示例（仅参考，需改写）：
```
针对问题X，本问需【要求】。基于【机理/目标约束】建立【方程/优化模型】，设定【参数/初始条件】后用【数值方法/求解算法】求解。【数值结果】，模型【评价】。
```

**③ 建议型**（基于前文结果给出建议/策略，一笔带过）——要素：依据 → 角度 → 建议数量 → 核心结论。示例（仅参考，需改写）：
```
针对问题X，本问需【要求】。结合前文结果，从【角度1】【角度2】给出【建议数量】条建议，核心是【要点】。
```

**一页验证（编译后必做）**：
```bash
PAGE=$(grep 'abstract:end' paper/main.aux | grep -oE '\{[0-9]+\}' | head -1 | tr -d '{}')
if [ "$PAGE" -gt 1 ]; then echo "摘要超出一页（第${PAGE}页），需缩减"; fi
```
若超页，按比例缩减各问题段字数后重新编译，直到 `PAGE=1`。

**摘要禁止**：公式、模型检验内容、超字数、摘要末尾加 `\newpage`、任何 `\textbf` 加粗。
**摘要数值必须来自真实求解结果**（求解阶段输出的 keyMetrics），不得编造。

**去 AI 味审查检测**（套话黑名单，命中需改写）：
```bash
grep -nE '综上所述|高效解决方案|提供了.*(决策依据|有力支撑)|具有重要的现实意义|随着.*的发展|提出了基于.*的系统解决方案' sections/abstract.tex
```

## 3. 引言规范

| 部分 | 格式 | 要求 |
|------|------|------|
| 问题背景 | 文字，2-3段 | 宏观背景→问题来源→当前困境→解决渴求，不分点 |
| 研究意义（可选） | 文字，1段 | 社会意义 + 科研意义 |
| 问题重述 | `问题N：`（不加粗）每问≤2行 | 问题本质缩写，删背景，**绝对不可照抄原文**，不加分点符号 |

## 4. 总体分析规范

- 三段式结构，不画流程图
- 第1段：一句话总体表达
- 第2段：针对问题X建立了XX来解决XX（逐问串联）
- 第3段：总结各问题之间的递进关系

## 5. 问题分析规范

- 每个子问题单独小节
- 包含：相关条件 → 整体分析 → 初步方法及理由
- **仅当流程图文件实际存在时**才引用思维流程图；图不存在则纯文字论述，不得引用不存在的图
- 体现"从具体到抽象"的思维过程，不含检验和结果

## 6. 模型假设规范

- 两段式：第1段"为简化问题，本文做出以下假设："
- **itemize 环境**，每条以 `假设N：` 开头（**不加粗**），≤2行
- **共 5-8 条**，覆盖 6 类：① 题目明确给出 ② 排除小概率事件 ③ 仅考虑核心因素 ④ 模型本身要求 ⑤ 参数分布假设 ⑥ 简化假设

## 7. 符号说明规范

- 三列表格（符号 | 说明 | 单位），**使用 longtable 格式**（见第 9 节）
- 仅收录重要变量，符号与公式、图表、代码变量名一致
- 以全局统一符号表为基准（多子问题符号冲突必须在写作前消解）
- **全局符号表（`notation.tex`）是唯一定义处**：建模/求解章节正文**不得重复定义符号表已有符号**（禁止再写"其中 $X$ 为……"等重复定义句）
- 公式中新出现且具全文意义的符号 → **补进全局符号表**后正文直接引用，不另立定义
- **保留（宽松档，非零说明）**：公式紧邻的**非定义性物理解读句**（取值范围/边界行为/检验结论，如"$\text{CR}<0.10$ 通过一致性检验""当 $A\to\infty$ 时 $\eta\to1$ 趋于饱和"）可少量保留以提升可读性

## 8. 模型评价/改进/推广规范

- **优点 4 条 + 缺点 2 条**，按模型写不按问题写，优点多于缺点
- itemize 格式，每条以 `优点N：` / `缺点N：` 开头（**不加粗**），每条≤3行
- 改进与推广：各一段自然段落，不分点
- 改进必须针对具体缺陷且可操作（如"将 Logistic 替换为 GAM"），禁止"可以考虑更复杂的模型"等空泛表述

## 9. 表格规范（全论文统一）

**所有表格**（符号说明、算法对比、模型结果、检验分析等一切表格）统一使用 `longtable` + `>{\centering\arraybackslash}p{}`，总宽=`\textwidth`，各列居中可跨页。

**三线表线宽（强制·全局钉死）**：`\toprule`/`\bottomrule`（顶线/底线，加粗）= **1.5pt**，`\midrule`/`\cmidrule`（栏目线/辅助线，不加粗）= **0.5pt**（顶/底加粗、栏目/辅助不加粗是三线表通行规范）。由预导言 `\setlength` 统一设置，单表不再手改：
```latex
\setlength{\heavyrulewidth}{1.5pt}
\setlength{\lightrulewidth}{0.5pt}
\setlength{\cmidrulewidth}{0.5pt}
```

**列宽公式（强制）**：各 `p{比例\textwidth}` 比例总和 = **1.04 − 0.04 × N**（N 为列数）：

| 列数 | 比例总和 | 示例分配 |
|------|----------|----------|
| 2列 | 0.96 | `{0.18+0.78}` |
| 3列 | 0.92 | `{0.16+0.16+0.60}` |
| 4列 | 0.88 | `{0.12+0.12+0.12+0.52}` |
| 5列 | 0.84 | `{0.12+0.18+0.18+0.18+0.18}` |
| 6列 | 0.80 | `{0.10+0.14+0.14+0.14+0.14+0.14}` |
| 7列 | 0.76 | `{0.10+0.11×6}` |
| 8列 | 0.72 | `{0.09×8}` |

**标准模板**：

```latex
\begin{longtable}{>{\centering\arraybackslash}p{0.18\textwidth}>{\centering\arraybackslash}p{0.78\textwidth}}
  \caption{表名（中文）}
  \label{tab:标签} \\
  \toprule
  列1 & 列2 \\
  \midrule
  \endfirsthead
  \caption{表名（续）} \\
  \toprule
  列1 & 列2 \\
  \midrule
  \endhead
  \bottomrule
  \endfoot
  内容 & ... \\
\end{longtable}
```

规则：`\caption{}` 在 `\label{}` 之前，`\label{}` 后紧跟 `\\`；`\endfirsthead/\endhead/\endfoot` 固定结构；`\endfoot` 含 `\bottomrule`（longtable 末页亦用 `\endfoot`，故单页表底线由它绘制），**表末不再重复写 `\bottomrule`**；格子文字一行显示；编译后检查 Overfull。

**三条铁律（最易跑偏，弱模型尤甚）**：① 每列必须 `>{\centering\arraybackslash}p{}` 居中，**禁裸 `p{}`/`l`/`r`/`c`**（左对齐参差是最常见错版）；② 列宽必须 `比例\textwidth` 且总和=1.04−0.04N（占满版心并居中），**禁 `cm`/`mm`/`pt` 绝对定宽**（会留白左靠、不满宽）；③ `\bottomrule` 全表**仅一条**且只在 `\endfoot`，表末不得再写（重复=双底线）。

**生成期保证（推荐·I3）**：结果表/符号表等由数据驱动的表，**优先用 `references/make_table.py` 的 `make_longtable(df, caption, label)` 从 CSV 直接生成** longtable 代码再粘入章节 `.tex`——结构、居中、比例满宽、单底线一次到位，不靠手写。
**审查期检测（强制·I1·双保险）**：终验前对 `paper/sections` 跑 `python references/check_tables.py paper/sections`——检 H1 裸列未居中 / H2 绝对单位定宽 / H3 比例非满宽 / H4 缺三线 / M1 双底线 / M2 头尾不全；存在 HIGH（退出码≠0）必修复重编。

## 13. 模型检验规范

> **🔴 完美指标即红灯（数据泄露硬护栏·问题A修复，所有预测/分类题强制）**：分类 `AUC≥0.99` 或 `F1≥0.99`、回归 `R²≥0.99`（含 5 折 CV 均值）一律视为 **疑似数据泄露/标签穿越**，**不是亮点而是缺陷信号**——真实预测任务几乎不可能达到完美分数。命中时必须：①复查特征是否含与标签**同期或未来**的信息（典型：用"当天评论/点赞"预测"当天关注"，特征与标签同时刻强共线即泄露）；②若确为泄露，改用**严格滞后特征**（仅用目标时刻之前的数据）重训，指标回落到合理区间才算有效；③若受赛题数据所限无法消除（如标签当日才产生），必须在正文与摘要**显式声明该局限并保守措辞**（用"受数据限制""仅反映同期相关性"等），**严禁把完美指标当作模型优势宣传**。审查检测：`grep -nE 'AUC[^0-9]*(1\.0|0\.99)|F1[^0-9]*(1\.0|0\.99)|R.?2?[^0-9]*(1\.0|0\.99)|=\s*1\.000' solve_data.md` 命中即强制人工复核。摘要里出现 AUC=1.000/F1=0.999/R²=0.999x 等且无局限声明 = critical。

检验维度**按题型选用**，不对不适用的模型强套指标：

- 灵敏度分析：控制变量法，至少 2 个核心参数（所有题型通用）
- 模型检验：使用前检验 + 使用后检验（所有题型通用）
- 【预测/回归】报告 $R^2$ 与调整 $R^2$（$R^2<0.6$ 须说明原因并给替代方案，如 GAM/NLME）+ 误差分析 MAE/RMSE，优先 5 折交叉验证
- 【分类】报告 AUC、灵敏度、特异度（AUC<0.8 须说明升级措施，如 XGBoost/LightGBM）；不得仅凭 p 值显著声称"拟合良好"
- 【优化/规划/几何】**不报 $R^2$/AUC**：报目标收敛曲线、多初值/多次重启的解稳定性、与 baseline 或理论界对比、数值积分步长 $\Delta t$ 的收敛性、约束满足验证；**用元启发式（遗传/差分进化/退火等）求解可线性化的 LP/MILP 时，必须报告最优性间隙**——以 LP 松弛上界或精确求解器（LINGO/Gurobi/intlinprog/PuLP）的解为标尺，给出 $\text{gap}=(\text{上界}-\text{启发式解})/\text{上界}$，gap 过大须加大迭代或改用精确求解器
- 【随机/风险优化】（随机规划、CVaR/VaR、鲁棒优化、益损值矩阵）：报告风险-收益权衡（有效前沿）、置信水平 $\alpha$ 与风险厌恶系数 $\lambda$ 的灵敏度、蒙特卡洛场景数 $N$ 增大时目标值的收敛、与确定性模型（用期望参数求解）的解对比；益损值矩阵须同时给 ≥2 种决策准则结果并说明取舍
- 【反演/拟合/机理】报拟合 RMSE 与 $R^2$、预测曲线与实测**叠加对比图**、关键参数的**置信区间**、病态性/正则化讨论
- 【微分方程/动力学机理】参数恒定时求解析解（特征根/拉普拉斯变换）、参数时变时求数值解（Runge-Kutta/`ode45`），**恒定情形下解析解与数值解必须数值吻合**（机理题最强的求解正确性验证）；并报守恒量核对、数值积分步长收敛性
- 【成分数据】（各分量和为定值的占比/浓度/构成数据）：建模前先做**闭合校验**（分量和≈100%，越界样本剔除/归一化）与**对数比变换**（CLR/ILR，零值先做零替换），**禁止对原始百分比直接算 Pearson 相关或做 PCA**（闭合约束导致伪相关）；报告变换前后结论一致性、零替换敏感性、预测成分的闭合性与实测对比
- 【评价/决策】权重扰动稳健性、排序稳定性（如权重 ±10% 排名是否反转）
- 【聚类/分段/信号】聚类数 $K$、有序分割段数、小波基与分解层数等**结构参数禁止拍脑袋**：聚类按轮廓系数/肘部法，有序分割按"段数-段内误差"拐点，小波按信号特征说明选择依据；分段结果须有业务可解释性，并与独立方法（如小波突变点 vs Fisher 分割点）互证
- 【时序】预测建模前先用 R/S 分析（Hurst 指数）判断长记忆性：$H\approx0.5$ 接近随机游走时长期预测不可靠，须说明并谨慎外推
- 医疗筛查等不对称代价场景：在灵敏度 ≥ 95% 硬约束下最大化特异度，禁止直接最大化 F1/准确率
- 相关性分析：数值变量注明 Pearson/Spearman 与显著性；**类别变量×类别变量**的关联（如"是否风化"与"类型/颜色/纹饰"）须用**卡方独立性检验**（小样本或期望频数<5 改 Fisher 精确检验/Yates 校正），不可用 Pearson/Spearman；多变量/多分组同时检验须做 Bonferroni 或 FDR 多重比较校正
- **不确定性量化**（测量误差显著或建议型结论必做）：将输入误差（如 $c\sim N(\mu,\sigma^2)$）传播到关键结论，用解析或蒙特卡洛/Bootstrap 给出结论的置信区间/鲁棒区间，而非单点
- **多方法交叉验证**（强可信度证明）：关键结果尽量用独立的第二种方法（或解析特例/极限情形）复算，数值一致即互相印证——如同一动力学量用 Runge-Kutta、`ode45`、解析解三种求解器得到量级一致的结果（2022A 波浪能三篇分别用三法均得最大功率 229~230W），或定位问题用代数解与计算几何解互证，是最有力的正确性佐证
- **数量级合理性核对**（必做底线）：关键结果须与国标/经验值/解析估计对照量级；若独立方法或不同子模型的同一结论**数量级不一致**（如同一抽样方案一处得 $n<100$、另一处得 $n>1000$），**必有一处建模有误，须定位并修正错误源**，绝不能把互相矛盾的两个值并列写进论文。**优化题专项**：目标值（利润/成本等）须以 LP 松弛上界或精确解为标尺自检，若元启发式解显著低于上界（gap 大）即判未收敛而非"模型差异"——同一道大规模 MILP 用不同算法得数倍悬殊目标值，几乎必是欠优解未收敛所致

## 14. 版式硬错判定（Phase D Layer1/2 可执行检测的唯一权威）

渲染后才暴露的版式硬错在 `.tex` 源码里看不出，须在终验编译成功后按下列确定性检测判级（模型无关，永远执行）：

- `Missing character: There is no ... in font`（main.log）→ **critical**：CJK 缺字/乱码，零容忍，必修复重编。
